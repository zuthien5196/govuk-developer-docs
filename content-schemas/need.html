Schema: need

Frontend schema

This schema describes the content item that is returned from the content store.

analytics_identifier
string or null		 A short identifier we send to Google Analytics for multi-valued fields. This means we avoid the truncated values we would get if we sent the path or slug of eg organisations.
base_path
string		 A path only. Query string and/or fragment are not allowed.
content_id
string
description
or null

object

boolean		 Whether all linked organisations meet this need

string		 Why the user wants to do it

array

string		 What the user wants to do
impact
string		 Impact of GOV.UK not doing this
justifications
array		 How this need fits in the proposition for GOV.UK
legislation
string		 Legislation that underpins this need
met_when
array		 Provides criteria that define when this user need has been met
need_id
string		 Six digit id which used to be the primary id for Needs. Still being displayed in Maslow and Info-Frontend, but likely to be deprecated in the future.
other_evidence
string		 Any other evidence to support this need, ie. user research, campaigns, user demand
role
string		 The type of user, such as a small business, a tax agent, a healthcare practitioner

integer		 Number of pageviews specific to this need generated each year

 All organisations linked to this content item. This should include lead organisations.

 The organisation that published the original version of the page. Corresponds to the first of the ‘Lead organisations’ in Whitehall for the first edition, and is empty for all other publishing applications.
